# 基于 node+Koa+mysql 实现后台管理 API 接口

## 错误类型

错误码由五位数构成，第一个数字表示前端还是后端错误，第二和第三位数字表示那个模块，第四和第五位表示具体的错误信息编号（见 config->errorType 文件）

例如：10001：表示错误属于前端的错误，用户注册模块的编号为 01 的错误

| 第一位的数字含义 |          |
| ---------------- | -------- |
| 1                | 前端错误 |
| 2                | 后端错误 |

| 第二和第三位的数字含义 |          |
| :--------------------- | -------- |
| 00                     | 用户模块 |
| 01                     | 权限模块 |
| 02                     | 部门模块 |
| 03                     | 菜单模块 |
| 04                     | 角色模块 |
| 05                     | 商品模块 |
| 06                     | 分类模块 |
